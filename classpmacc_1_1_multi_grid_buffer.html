Detailed Description

template<typename Type_, unsigned DIM, class BufferNames, class BORDERTYPE = Type_>
class pmacc::MultiGridBuffer< Type_, DIM, BufferNames, BORDERTYPE >

GridBuffer represents a DIM-dimensional buffer which exists on the host as well as on the device.

GridBuffer combines a HostBuffer and a DeviceBuffer with equal sizes. Additionally, it allows sending data from and receiving data to these buffers. Buffers consist of core data which may be surrounded by border data.

Template Parameters
Type_datatype for internal Host- and DeviceBuffer
DIMdimension of the buffers
BufferNamesa class with a enum with the name "Names" and member with the name "Count" with number of elements in Names etc.: struct Mem { enum Names{VALUE1,VALUE2}; static constexpr uint32_t Count=2; };
BORDERTYPEoptional type for border data in the buffers. TYPE is used by default.
